# 日历看板系统

这是一个基于H5前端和Excel数据库的动态日历看板系统，用于管理和展示工况时间数据。

## 功能特点

1. **日历视图展示**：以月历形式展示任务和工况信息
2. **数据管理**：支持添加、编辑、删除任务
3. **Excel集成**：支持从Excel导入数据和导出数据到Excel
4. **数据验证**：自动验证每天的任务时间总和是否为24小时
5. **多级时效分类**：支持三级时效分类（生产时效/非生产时效）
6. **项目筛选**：可按项目名称筛选显示任务

## 文件结构

- `index.html` - 主页面文件
- `styles.css` - 样式文件
- `script.js` - 功能实现脚本
- `template.html` - Excel数据模板生成器
- `README.md` - 项目说明文档

## 使用方法

1. **打开日历看板**：
   - 直接在浏览器中打开 `index.html` 文件

2. **查看日历**：
   - 使用顶部控制栏选择年份和月份
   - 可以选择特定项目进行筛选
   - 点击刷新按钮更新数据

3. **管理任务**：
   - 点击日期单元格添加新任务
   - 点击已有任务编辑任务信息
   - 在编辑窗口中可以删除任务

4. **Excel数据操作**：
   - 点击"导出Excel"按钮导出当前数据
   - 点击"导入Excel"按钮从Excel文件导入数据
   - 打开 `template.html` 生成标准Excel数据模板

## 数据字段说明

- **工况名称**：任务的名称描述
- **工况一级时效**：生产时效/非生产时效
- **工况二级时效**：根据一级时效联动选择
- **工况三级时效**：根据二级时效联动选择
- **开始时间**：任务开始时间（24小时制，格式：HH:MM）
- **结束时间**：任务结束时间（24小时制，格式：HH:MM）
- **负责人**：任务负责人姓名
- **日期**：任务执行日期（格式：YYYY-MM-DD）
- **项目**：所属项目名称

## 数据验证规则

系统会自动验证每天的任务时间总和是否为24小时，如果不等于24小时，日期数字会显示为红色并提示警告。

## 技术实现

- **前端框架**：纯HTML5 + CSS3 + JavaScript
- **Excel处理**：使用SheetJS (xlsx)库处理Excel文件
- **数据存储**：使用localStorage进行本地数据存储
- **响应式设计**：支持不同屏幕尺寸的设备

## 注意事项

1. 首次使用时，系统会自动生成一些示例数据
2. 数据保存在浏览器的localStorage中，清除浏览器数据会导致数据丢失
3. 建议定期导出Excel文件进行数据备份
4. 导入Excel数据时，请确保文件格式与模板一致
